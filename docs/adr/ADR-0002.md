# ADR-0002

## Title

Business Case Reference Number

## Status

Accepted

## Context

Users need a stable and readable way to identify Cases in the interface.

UUID is a good internal technical identifier, but it is not suitable as the main identifier shown to users.

## Decision

AI Office keeps UUID as the internal identifier of `Case`.

Users see a business reference number in the format `CASE-000001`.

The numeric value is allocated in Application through `CaseNumberProvider`.

Persistence stores only the integer value of the reference number.

Formatting belongs to the presentation layer.

## Consequences

- UUID remains useful for internal links and system identity.
- Users work with readable and memorable case references.
- Number allocation can change later without changing Domain.
- Repository persists assigned numbers, but does not generate them.
- Gaps in numbering after a failed save are consciously accepted to keep number allocation simple and stable across restarts.
